Renderización en HTML, CSS y JavaScript

¿Cómo y cuándo se renderiza una página web?

1. ¿Qué es Renderizar?

Imagina que tienes una receta de cocina en papel. Para que puedas comer el pastel, alguien debe seguir la receta y prepararlo.

En términos de desarrollo web:

  • El código HTML, CSS y JavaScript es la "receta".
  • El navegador es el "chef".
  • El resultado en pantalla es el "pastel".

Renderizar es el proceso de convertir el código en algo visual y funcional.

Ejemplo de Renderización

Si escribimos este código en una página web:

<h1>Hola Mundo</h1>

El navegador lo renderiza mostrando "Hola Mundo" en la pantalla.

Si luego ejecutamos este JavaScript:

document.querySelector("h1").innerText = "Hola, Javier!";

El navegador re-renderiza el texto con el nuevo mensaje.

2. Renderización en el Cliente vs. Servidor

Renderización en el cliente:

  • El HTML es cargado primero.
  • El JavaScript obtiene y renderiza los datos dinámicamente.
  • Ejemplo: React, Vue.js.

Renderización en el servidor:

  • El servidor genera el HTML ya procesado.
  • El cliente recibe la página lista para mostrarse.
  • Ejemplo: PHP, Next.js.

3. Renderización Incremental

Algunas aplicaciones web modernas implementan técnicas de renderizado incremental para mejorar la velocidad:

  • Lazy Loading
  • Hydration (para aplicaciones React y Vue)
  • SSR con prefetching de datos

4. Optimización de Renderizado

Para mejorar el rendimiento:

  • Usar requestAnimationFrame() para animaciones fluidas.
  • Evitar cambios frecuentes en el DOM.
  • Aplicar técnicas de memoización y caching.
  • Utilizar Web Workers para procesos pesados.
